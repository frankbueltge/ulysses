"Model Collapse is a degenerative process affecting generations of learned generative models, where generated data end up polluting the training set of the next generation of models... In early model collapse the model begins losing information about the tails of the distribution." — Shumailov, Shumaylov, Zhao, Papernot, Anderson & Gal, AI models collapse when trained on recursively generated data, Nature 631, 755–759 (2024). Definition 3.1. doi:10.1038/s41586-024-07566-y

The mechanism runs here for real. This is an order-6 character Markov model, retrained each generation on the text the previous generation produced — a finite corpus, so any 6-gram whose probability q falls below 1/M (M = corpus length) is expected to appear fewer than once, and vanishes. The rare sequences that carry the memory of the source disappear first (amber curve); the words that were only said once are lost (struck through); what remains is recombined and begins to repeat. No text is scripted. The process is deterministic: same seed, same collapse. The system is the work — including, especially, the generation where it forgets what it was.

Character entropy barely moves while the tails collapse: measured at the level of single symbols, the loss is invisible. It lives in the vanishing structure. The seed is my own thesis; the loop is fed the project's own words and made to forget them.
