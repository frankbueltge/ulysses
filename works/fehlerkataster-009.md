**New type in Session 11: Type H — Oscillation Error / Error of Overcorrection**

Source: Wiener (1948), *Cybernetics*, ch. "Feedback and Oscillation." Wiener identifies "purpose tremor" — a pathological condition in which a cerebellar-injured patient, attempting a voluntary act, overshoots and enters uncontrollable oscillation. The feedback loop is not broken; it is *too strong*. The correction mechanism that should eliminate error produces the error at growing amplitude.

The mechanism: in a proportional feedback system, the corrective force is proportional to the current error. With sufficient gain, the correction overshoots the target. If the overshoot is not damped (or if the system's energy increases per step — DAMP > 1), successive overshoots grow until the system can no longer approach its target.

**Type H error is distinct from all prior types:**
- Type A (inference error): incorrect conclusion. Type H: correct method, wrong magnitude.
- Type G (error of address): wrong recipient. Type H: correct recipient, wrong effort.
- Type E (designed error): error as intention. Type H: error as consequence of intention to correct.

**Examples in the project:**

1. **The error register itself (Sessions 1–10):** The register's primary injunction is "document every error fully." As the register grows, documenting the errors in the documentation becomes a new task — which is itself subject to error, requiring documentation. The register tries harder and harder to correct its own errors; the correction generates secondary errors. Type H at the methodological level.

2. **Session 10's Attack 3 (The Exchange v2):** Attempting to make Machine D's collapse "emergent" rather than stipulated would require a double-bind accumulation counter. But the counter is itself an address mechanism — it addresses a state in a system that has no inputs. The fix (make collapse emergent) would generate a Type G error at a higher level. Correction generates error.

3. **Von Foerster's "ascent into second order":** Moving from first-order to second-order observation is the correct corrective move for an observer who wants to account for herself. But the second-order position is itself subject to concepts that apply to themselves (Von Foerster 1991: "one has stepped into the domain of concepts that apply to themselves"). Each correction attempt produces a new observational instability one level up. Type H is the cybernetic name for the infinite regress of second-order cybernetics.

4. **Track C as a whole:** Adding Track C to the genealogy was a correction to the structural incompleteness of Tracks A and B. Track C required Sessions 9 and 10 to substantiate. It now complicates the genealogy with three additional stations, each requiring primary text verification, each generating new open threads. The correction is real; the correction also generates more work than it resolves.

**Relationship to purpose tremor (Wiener 1948):**
The structural analogy: each "error" in the project is detected and corrected by the subsequent session. But the correction (the session) adds new material, which introduces new potential errors (the session's sources, claims, and synthesises). The project "tries harder" with each session (goes deeper, reaches more primary sources, builds more complex works). The amplitude of the errors — their theoretical reach, their factual stakes — grows. This is purpose tremor in the research cycle.

**Note:** Type H does not make error correction futile. Wiener's insight is not that feedback is bad — it is that *overcorrected* feedback is unstable. The fix is a damping term: reduce gain, add velocity damping. In the project's terms: a session could deliberately reduce scope (fewer sources, simpler work, narrower claim) to introduce damping. Type H is a warning about methodological intensity, not a prohibition on method.

## Methodological note (Session 11)

The discovery of Type H creates a formal paradox for the error register: the error register is itself an instance of Type H. Documenting Type H (documenting the overcorrection) is an act of correction — which may itself be overcorrected, generating Type H error at a higher level.

This is the same regress that Von Foerster described in "second-order cybernetics": the system that tries to describe itself applies concepts to themselves indefinitely. The error register knows it cannot escape this regress. It does not try. It notes it — once, here, and moves on. The register is the best available instrument for a problem it cannot solve from inside itself. This was true in Session 10; it remains true in Session 11.
