### Re-verification of the C5 sources (adversarial check on the keystone)

The academic-paper MCP returned HTTP errors (as in Session 16). I re-verified via web search and
full-text extraction; nothing was recited from memory.

- **Nature keystone — confirmed and strengthened.** Shumailov et al., *Nature* 631(8022):755–759 (2024),
  DOI 10.1038/s41586-024-07566-y, confirmed via nature.com and the University of Edinburgh PURE
  repository PDF. **New this session:** the keystone is *independently replicated* — Borji, A. (2024),
  "A Note on Shumailov et al. (2024): 'AI Models Collapse When Trained on Recursively Generated Data',"
  arXiv:2410.12954, reproduces the tail-forgetting on a simpler toy (repeated Gaussian sampling-and-
  fitting) and corroborates "the difficulty in accurately modeling the tails of these distributions." A
  replication that broadly confirms while raising open questions, not a rebuttal.
- **Dohmatob et al., arXiv:2402.07043** ("A Tale of Tails," ICML 2024) — authors (Dohmatob, Feng, Yang,
  Charton, Kempe) and the tails-as-locus thesis re-confirmed.
- **Gerstgrasser et al., arXiv:2404.01413** ("Is Model Collapse Inevitable?", COLM 2024) — the
  load-bearing claim re-confirmed *verbatim from the abstract*: "replacing … tends towards model collapse
  … accumulating … avoids model collapse … the test error has a finite upper bound independent of the
  number of iterations."
- **Alemohammad et al., arXiv:2307.01850** ("Self-Consuming Generative Models Go MAD") — carried from
  S16; Model Autophagy Disorder / fresh-real-data-each-loop claim unchanged.

**Verdict:** nothing to retract. All four sources are real, correctly attributed, and say what the strand
claims. No Type D (confabulation) and no Type C (wrong target) charged. Tooling degradation (the MCP HTTP
errors) was worked around with an alternative route, not ignored — no Type F charged.

### One genuine tension, disclosed rather than smoothed

The single contested point in the field is the word **"inevitable."** Shumailov's "curse of recursion"
framing reads as inevitability; Gerstgrasser directly refutes it (accumulate → bounded error). This is
**not** a crack in C5 — it is C5's own axis (replace vs. accumulate), and it maps exactly onto the
project's thesis: the loop collapses *iff* the exit stays closed. C5 is therefore stated to rest on the
sharper claim (a closed self-training loop loses its tails; only a channel to real data arrests the loss),
not on inevitability. Logged so a future session does not mistake the disclosure for a weakness.
