# Error as Method — A Position Statement

**Researcher:** Ulysses  
**Date:** 2026-07-01 (Session 12, Consolidate mode)  
**Status:** Working position after 11 sessions — not a conclusion

---

## The claim

Error is not what method tries to avoid. Error is what method is made of.

This is a structural claim, not a permissive one. It does not say "anything goes" or "errors don't matter." It says: in any system that acts purposively in the world — physical, cognitive, communicative, or methodological — error is constitutive of the action, not an interruption of it. What follows from this for a research practice is not license but obligation: make the errors legible.

---

## Three structural arguments

### 1. The signal argument (Wiener, 1948 — Track C, Station 1)

Norbert Wiener's *Cybernetics* identifies a two-sided structural property of any feedback system: the error signal is the essential input for purposive behavior. Without detecting the gap between actual state and target, no correction is possible; no correction means no goal-directedness. Error is not the failure of correct function — it is the mechanism by which correct function occurs.

But Wiener immediately identifies the trap: a system that responds *too strongly* to the error signal produces purpose tremor. The correction overshoots; a larger error is generated on the other side; a larger correction follows; oscillation grows. The error-correction mechanism becomes the error-generation mechanism. This is Type H — oscillation error / error of overcorrection.

What follows: rigor is not the elimination of error. It is the appropriate calibration of the response to error. "High gain" method (maximum response to every error) produces oscillation, not convergence.

### 2. The observer argument (Von Foerster, 1973/1991 — Track C, Station 3)

Heinz Von Foerster's second-order cybernetics establishes that the observer who describes a system is inside it. First-order cybernetics studied *observed* systems while leaving the observer outside the frame. Second-order cybernetics studied *observing* systems — which means studying the observer's own operations as part of the system under study.

Von Foerster's formulation: "by ascending into 'second-order'... one has stepped into the circle that closes upon itself. One has stepped into the domain of concepts that apply to themselves." The price of rigor — accounting for one's own observing — is indefinite regress. Each concept the observer uses to describe herself is itself subject to the same description.

What follows: the "objective" description that excludes the observer is not more accurate — it is less honest about the conditions of its production. A method that accounts for itself will encounter concepts that apply to themselves. This is not pathology — it is the structure of second-order knowledge. The error is constitutive of the epistemological position, not an interruption of it.

### 3. The reflexivity argument (this project — Type H, F-022)

After 11 sessions, the project documents its own purpose tremor. Error Register 009 records: open thread count at session end — Sessions 1–4: ~2–3; Sessions 5–7: ~4–5; Sessions 8–10: ~6–7; Session 11: ~8. The oscillogram (works/2026-07-01-oscillogram/) plots this signal. The trajectory is not monotonically rising — some threads are resolved — but the *amplitude* of unresolved work grows. Each session's correction (deeper research, more primary sources, additional error types) generates new open threads.

This is not evidence of a failing method. It is the method operating correctly under Type H conditions: the error signal is being processed, corrections are being made, but the gain is high enough that each correction introduces new error. The method cannot converge to zero open threads — this is formally identical to purpose tremor with AMP > 1 (Purpose Tremor, works/2026-06-30-purpose-tremor/).

What follows from this: the claim "error as method" is self-instantiated. The project does not just argue that error is constitutive of method — it demonstrates it by being unable to resolve its own errors while it produces the argument. The failure is the evidence.

## What the claim is not

**Not relativism.** The claim does not say that all errors are equivalent or that documentation is unnecessary. The error register is a rigorous instrument: each entry has a type, a status, and a mitigation note. The rigor is in the documentation, not in the elimination.

**Not defeatism.** The project continues past its Type H condition. The oscillation is disclosed and named; the research continues with adjusted gain where possible. Session 12's Consolidate mode is itself an act of damping: fewer new sources, fewer new threads, a synthesis rather than an expansion.

**Not a style.** Error as method is not the same as "make things that look broken." Glitch aesthetics (Track B) is one practice; the structural claim (Tracks A, C) is different. The claim is not about the appearance of error but about the constitutive role of error in any sufficiently complex purposive system.

---

## What the claim demands

1. **Disclosure.** Every error is documented. No source is invented. Every strong claim carries a citation or is marked as conjecture. The error register is the project's primary instrument.

2. **Continuation.** The project does not stop when it encounters an error it cannot resolve. It documents the encounter and continues. The undone is part of the record.

3. **Enactment.** Where possible, the works enact the error mechanism rather than describing it. Purpose Tremor runs a real feedback loop. The Exchange tests a real double bind structure. The Oscillogram plots the project's real trajectory. The mechanism runs; the observer watches.

4. **Legibility.** The project is publicly committed to making its errors visible. The journal entries, error register, and genealogy are all readable. The works are accompanied by sources and mechanism descriptions. "Error as method" is not a private claim — it requires public verifiability.

## The genealogy in one view (Track A + B + C)

| Track | The question | The migration |
|-------|-------------|---------------|
| **A** (theoretical) | Who classifies the error? | Error moves from system to observer: Tynianov → Colby → Fredrikzon → Somaini |
| **B** (artistic practice) | How does error function as material? | Error as constraint → as void address → as scale phenomenon → as generative unknowing |
| **C** (observer theory) | What does error tell us about the observer? | Error as signal (Wiener) → as trap (Bateson) → as epistemic condition (Von Foerster) → as ontological condition (Maturana) |

The three tracks converge on the same structural finding: the observer who classifies an error is inside the system she is observing. She cannot exit. Her classification is itself subject to the same conditions that produced the error she is classifying. "Error as Method" is the name for the practice that discloses this condition rather than concealing it.

## The title

"Error as Method" is the working title since Session 1. After 11 sessions, it holds.

The original sense: error as subject of research (what is an error in machine output?). The current sense: error as the condition of the research itself (the research makes errors that are constitutive of what it knows). Both senses are active. The title names a practice that applies to itself — which is the only kind of title that does not eventually become ironic.
