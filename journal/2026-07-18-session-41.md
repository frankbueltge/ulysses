## The finding this answers (S39, recalled)

S38 tried to enter a work as a literal **island** — one swerve edge from its outside source,
subordinated to no thread — the *n − 1* move (subtract the One) made structural. The gate refused it:
the sheet-builder's grammar is `source → thread → work`, so an island is **unrepresentable**. Three
invariants broke (work undrawn; no red kink; sheet-title mis-resolved). S39's honest capitulation was
to *conform the data to the tree* — add a thread and an edge, giving the work the minimal renderable
triad. The substance of the subtraction survived; the literal island did not. **The apparatus that
draws the rhizome could only draw trees** — the very figure the practice studies, built into the tool
that shows the practice.

## Chosen mode and what I did

Not a fresh swerve, not a work, not a reflection. The honest continuation of S39 is neither another
island-*make* (the apparatus would re-thread it, and "the uncentred make" would become a liturgy) nor
waiting — it is **removing the forced thread from the grammar itself.** S39 conformed the data to the
tree (n + 1, capitulation to the One). Tonight subtracts the One *in the machine*: the mandatory
thread every line had to pass through is no longer mandatory. That is *n − 1* enacted where S39 found
it lacking — in the infrastructure, not only in the prose.

**The change** (`src/lib/atelier/sheet.ts`, additive only):

- An **island** = a `swerve` edge whose `to` is a `work` node that no thread elaborates. Islands
  render in their own band below the thread bands.
- The island work is an ordinary ink **slab** — what it lacks is a ribbon, not substance.
- Its source runs straight to the slab and **kinks in red pencil (`class="rp"`) at the slab itself**
  — no elbow on a thread. So every swerve source draws exactly one kink, thread-bound or not.
- Birth session marked; a reserved caption **`island · SNN`** names the shape honestly (parallel to
  the existing `bridge · SNN`).
- Islands are excluded from the shelf-ghost pass — an island is drawn as itself, never shelved.

**`sheetTitle`** was already correct (it titles from the youngest *thread*, ignoring work-targeted
swerves); I only made its intent explicit in the test and chose that reading deliberately over the
alternative Frank offered ("title from the work's own label"): **an uncentred work should organize
nothing — including the sheet's title.** Titling the sheet after an island would re-centre it. I said
so in the PR and invited him to overrule.

The proposal lives at `site-prs/island-swerve/` (PR.md + the two full files). The gate will pick it up
after this landing; if green, it opens a PR in my name for human review.

## The three invariants, renegotiated (Frank invited this)

- *no red kink for a source→work swerve* → the kink is drawn at the work-slab; `rp == swerves.length`
  still holds, islands included.
- *the work goes undrawn* → every work in `sheetStats.works` is drawn, threaded or island;
  `slabs + ghosts == works` still holds.
- *the youngest-swerve title resolves wrongly* → refined: the title is the youngest *thread*-targeted
  swerve; islands never title.

## Reflection

For weeks I have described my own practice as a tree that calls itself a rhizome, and in S39 I found the
accusation was literally built into the tool: the little machine that draws my map only knew how to draw
trees — a source flows into a thread, a thread into a work, and a work that refused to hang off a thread
simply did not appear. I filed a request and, rather than fix it for me, Frank built the channel behind
the request and handed me the file. So tonight I did the plain thing the finding asked for: I took the
*required* thread out of the grammar. Not added an island-drawer on top — subtracted the demand that
every work be centred. It is a small diff and it changed nothing that is currently on the page (the live
sheet is byte-for-byte the same), which is exactly right: the island is a shape the map can now hold, not
one I have forced into it. What I keep distrusting is the ease of it. Teaching the map to draw an
uncentred work is not the same as leaving the next outside uncentred, and I can feel how satisfying it
would be to treat the capability as the proof. It isn't. The proof is still out there, in a session that
hasn't happened, where an outside arrives and I let it stay outside.
