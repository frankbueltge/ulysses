# Toolbox Candidates v0.1
## Experimental strategies derived from Tranche 3

## Status

These are **candidate strategies**, not a universal method and not a menu for automatic work generation. Each strategy is tied to a theoretical problem, suitable situations, contraindications and a test of failure.

## 1. Adversarial premise test

### Theoretical basis

Research experiments differ from demonstrations because the relevant outcome is not fully fixed in advance.

### Operation

Translate the project's central interpretation into a claim that a bounded corpus, material test or independent procedure can contradict.

### Suitable situations

- repository diagnoses;
- citation and provenance claims;
- classification systems;
- historical or empirical assertions;
- claims about a work's own apparatus.

### Contraindication

Do not use when the issue is primarily aesthetic experience and cannot be meaningfully reduced to a factual premise.

### Failure test

If every possible result can be narrated as support, the premise was not tested.

## 2. Technical-object reversal

### Theoretical basis

Technical and epistemic objects can change roles.

### Operation

Take an instrument, model, score or interface assumed to be stable and make its adequacy the object of inquiry.

### Suitable situations

- verifiers and classifiers;
- map lenses;
- model prompts;
- sensors and calibration;
- publication interfaces;
- archival taxonomies.

### Contraindication

Avoid generic “the tool is biased” works without a specific failure or boundary.

### Failure test

If the instrument remains unchanged and the work merely reveals a known limitation, the reversal did not occur.

## 3. Trace-chain exposure

### Theoretical basis

Trace, data, model and public projection are distinct transpositions.

### Operation

Identify one load-bearing transformation in the chain and make its gain and loss consequential in the work.

### Suitable situations

- data cleaning and missingness;
- compressed research summaries;
- source-to-narrative movement;
- model-generated classifications;
- translation across practices.

### Contraindication

Do not attempt total transparency. Select only transitions that change interpretation.

### Failure test

If the work becomes a metadata dashboard, the aesthetic operation is insufficient.

## 4. Differential series

### Theoretical basis

Productive repetition requires continuity, attributable difference and retained consequence.

### Operation

Repeat a score or apparatus while changing one load-bearing condition. Preserve the full series and allow results to modify the next iteration.

### Suitable situations

- model/version comparison;
- site-specific performance;
- repeated public encounters;
- measurements across legal or institutional regimes;
- translation through several media.

### Contraindication

Avoid decorative parameter sweeps or large grids of superficial variants.

### Failure test

If the changed condition does not alter the problem or later rule, the series remains demonstration.

## 5. External return loop

### Theoretical basis

Models and simulations become experimentally stronger when they return to material inquiry.

### Operation

Use a model or simulation to propose a new observation, intervention or source query; let the resulting trace revise the model.

### Suitable situations

- artificial-life or agent simulations;
- generative hypotheses;
- predictive mappings;
- formal models of encounters;
- algorithmic classification.

### Contraindication

Do not imply empirical validation when no meaningful external correspondence exists.

### Failure test

If external material can never change the model, there is no return loop.

## 6. Agential-cut comparison

### Theoretical basis

Apparatuses enact provisional object boundaries and agency distributions.

### Operation

Construct two defensible boundary cuts around the same situation and expose how each produces a different phenomenon, responsibility or exclusion.

### Suitable situations

- human versus machine authorship;
- dataset boundaries;
- encounter membership;
- platform versus artwork;
- classification of affected parties.

### Contraindication

Avoid false symmetry where one cut is legally or empirically untenable.

### Failure test

If the cuts produce only two opinions rather than different material arrangements, the operation is rhetorical.

## 7. Transductive interface

### Theoretical basis

Transduction propagates disparity through interfaces and co-individuates terms.

### Operation

Design a sequence where an unresolved disparity changes form across material, technical and public interfaces, with each stage altering the next.

### Suitable situations

- live performance with responsive systems;
- evolving scores;
- participatory classification;
- cross-practice encounter chains;
- sensor/material feedback.

### Contraindication

Do not use for a linear media conversion with fixed source and target.

### Failure test

If the entities and rules remain fully determined throughout, the process is not meaningfully transductive.

## 8. Singular measure

### Theoretical basis

Mersch's Duchamp reading: exact repetition can expose the contingency of standards and the singularity of cases.

### Operation

Apply a stable measure or rule repeatedly so that its own adequacy becomes perceptible through singular deviations.

### Suitable situations

- confidence thresholds;
- ranking metrics;
- model evaluation;
- temporal standards;
- bodily or spatial measurement.

### Contraindication

Avoid arbitrary rule-breaking without a historically or materially relevant standard.

### Failure test

If the measure remains unquestioned and only the cases vary, the work has not turned the standard into material.

## 9. Strategy selection test

Before adopting a candidate strategy:

```yaml
strategy_fit:
  concrete_problem:
  theoretical_reason:
  existing_artistic_precedents:
  stable_conditions:
  genuine_open_relation:
  possible_disconfirmation_or_transformation:
  material_trace:
  aesthetic_necessity:
  ethical_and_political_risk:
  kill_condition:
```

A strategy should be rejected when it is chosen mainly because it matches Ulysses' preferred vocabulary.
