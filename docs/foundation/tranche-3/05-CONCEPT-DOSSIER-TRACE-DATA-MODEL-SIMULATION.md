# Concept Dossier: Trace, Data, Model and Simulation

## 1. The transposition chain

Rheinberger describes experimentation not as direct access to a ready-made object but as a sequence of material and representational transpositions:

```text
phenomenon / interaction
        -> trace
        -> data
        -> model
        -> simulation
        -> possible return to experiment
```

Each transition creates new capacities. Each also introduces exclusions, conventions and losses.

For digital artistic research this chain is crucial because software systems often collapse all stages into one interface. A value appears on screen as if it were the phenomenon itself, while sensing, formatting, cleaning, selection, modelling and rendering disappear.

## 2. Trace

A trace is a material manifestation produced through an interaction between apparatus and research object. It is neither a transparent copy nor yet a fully interpreted representation.

Examples include:

- a sensor signal;
- a version diff;
- a failed request;
- a timing distribution;
- a changed file;
- an audio recording;
- a physical mark;
- a model completion under a specified configuration;
- a refusal or correction recorded in an encounter history.

The trace is indexical only in a qualified sense. It points to an interaction, but not to a pure origin independent of that interaction.

For Ulysses, the trace should be preserved before it is reduced to a narrative such as “the model drifted” or “the system discovered a contradiction.”

## 3. Data

Data are not simply collected traces. They are traces made durable, portable and comparable through formatting and selection.

Data production may involve:

- choosing what counts as an observation;
- assigning identifiers and timestamps;
- cleaning and excluding records;
- translating qualities into categories;
- aggregating or normalising values;
- deciding how absence is encoded;
- preserving or discarding context.

This transposition is productive: without it, traces may remain incomparable. It is also reductive: material conditions and ambiguities can disappear.

A Ulysses research trace should therefore distinguish at least:

```text
raw trace
selected record
derived feature
interpretive label
public projection
```

## 4. Model

A model condenses relations among data and offers a provisional way of seeing or acting. It can be conceptual, mathematical, diagrammatic, computational or spatial.

Models do not merely simplify an already known object. They can suggest new interventions, make previously disconnected traces comparable and generate hypotheses.

But a model is dangerous when:

- its categories become the object's supposed natural structure;
- its outputs are treated as observations;
- its convenience determines the research question;
- model fit is confused with explanatory adequacy;
- the public interface hides the modelling decisions.

In artistic research, a model can also be an aesthetic operator. That does not exempt it from declaring what it selects and excludes.

## 5. Simulation

A simulation executes a model and can produce model-data. It may reveal consequences that were not mentally anticipated by the designer. This can be genuinely useful.

Yet a simulation usually first tells us what follows from:

```text
the model's ontology
+ its initial conditions
+ its update rules
+ its parameters
+ its implementation
```

It becomes evidence about an external phenomenon only through additional argument and validation.

### Four simulation claims to distinguish

1. **Demonstrative claim** — the simulation makes a known mechanism perceptible.
2. **Exploratory claim** — it reveals consequences or regimes inside the model.
3. **Comparative claim** — variants help locate dependencies on assumptions.
4. **World-directed claim** — it supports an assertion about an external process.

The fourth requires the strongest external grounding. Ulysses has often moved too quickly from 1 or 2 to 4.

## 6. The return loop

The chain becomes experimentally productive when model or simulation returns to material inquiry:

```text
trace -> data -> model -> new intervention -> new trace
```

The return may:

- change what is measured;
- expose an instrument's blind spot;
- demand new source material;
- revise a category;
- distinguish competing explanations;
- terminate an untenable project.

Without such a return, a simulation may remain a compelling self-contained artwork or conceptual model. That can be artistically valid, but the epistemic claim must be limited accordingly.

## 7. Artistic consequences

A work can make the transposition chain perceptible without turning into a dashboard. Possible operations include:

- letting different stages contradict each other;
- making data loss materially consequential;
- exposing how a trace changes when made portable;
- requiring the visitor to choose a classification and experience its exclusions;
- placing raw material and model output in an unresolved relation;
- returning a public interpretation into the apparatus so that it changes later traces.

The aim is not total transparency. It is to make load-bearing transformations encounterable.

## 8. Minimum provenance structure

```yaml
trace_chain:
  phenomenon_or_interaction:
  apparatus:
  raw_trace:
  selection_and_cleaning:
  derived_data:
  model:
  simulation_if_any:
  public_projection:
  losses_and_exclusions:
  return_to_material:
```

Not every field must be public in full. Each project must decide which transitions materially affect interpretation and expose them accordingly.

## 9. Ulysses audit implication

*Differential Reproduction* is strongest when read as an exploratory simulation of a defined artificial world. It becomes weaker when internally defined novelty events are treated as epistemic things without an external return loop.

By contrast, *The Third Pile* and *Inaccurate Citations* preserve a stronger trace chain:

- repository or source records exist before the interpretation;
- a method transforms them into inspectable findings;
- the findings can contradict the starting diagnosis;
- primary material can correct the model's reading.

The lesson is not that simulations are invalid. It is that their epistemic scope must be stated precisely and their relation to external material must not be implied by aesthetic rhetoric.
